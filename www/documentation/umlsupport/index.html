Known Incompatibilities

Here is a list of features which are not implemented yet and can be seen as a violation of the UML specification (1.3).

All Diagrams

  • Qualifiers are missing.
  • Optional directional triangle missing on association names.
  • List of provided multiplicities is not complete.

Class Diagram

  • Association classes are missing. Though ArgoUML can read association classes from XMI files, it cannot display them correctly. That is due to a flaw in GEF, which needs a major rework.
  • Parameterized classes are missing
  • Packages can have stereotypes, but they are not displayed
  • Notes to any model element not implemented
  • Cannot add a method to an operation.
  • Cannot show collaborations on a class diagram.

Object Diagrams

  • Currently not implemented, use a Deploymentdiagram
  • Attributes cannot have attribute values

Activity Diagrams

  • Swimlanes missing

Sequence Diagrams

  • Missing completely from ArgoUML 0.14. (They were present in 0.10.1)

State Diagrams

  • Regions are drawn and handled as ordinary composite states.
  • There is no distinction between kinds of Actions.
  • There is no distinction between kinds of Events.
  • A "junction" is only partially supported.

Useful Optional Features

Class Diagram

  • Cannot specify package visibility as defined by Java.
