E.3. Suggested Manual Structure

Here are my thoughts. I think the user manual is really a set of two books, the tutorial manual (corresponding to Chapters 1–3 of the current manual), and the reference manual (Chapter 4 of the current manual)

I suggest that the tutorial book be based around an OOA&D process (any preferences), and that each UML concept is introduced with each step of the process, followed by an explanation of how to do it under ArgoUML. A simplecase study will be needed throughout. A lot of this material can come from existing Chapters 1–3, with some additional material from chapter 4 of the Cookbook, answers in the FAQ and Jason Robbins dissertation.

E.3.1. Tutorial Manual Structure

  1. Introduction

    1. Origins and overview of ArgoUML

    2. Scope of the User Manual. Include cross-reference to other documentation (Cookbook, FAQ, Quick Guide, online help, ArgoUML website etc).

    3. Overview of the User Manual. Explains that ArgoUML will be explained in the context of an OOA&D process, and with an example running through.

    4. Assumptions. At this stage assume the user knows OOA&D, but not UML.

  2. UML Based OOA&D

    1. Background to UML—what it is, history etc.

    2. UML based processes for OOA&D

    3. ArgoUML Basics—projects, drawing, exploring, details

    4. What ArgoUML has that other tools are missing (critics, to-do list, based in cognitive psychology theory).

    5. The Case Study

  3. Requirements Capture

    1. Use Case Diagrams (this section will be relatively large, because its the first time we use ArgoUML to create something).

  4. Analysis

    1. Concept Class Diagrams

    2. System Sequence Charts and Collaboration Diagrams

    3. System State Diagrams

  5. Design

    1. Class Diagrams for Realisation

    2. Sequence Charts and Collaboration Diagrams for Realisation

    3. State Diagrams for realisation

    4. Package Diagrams

  6. Build

    1. Deployment Diagrams

    2. Code Generation in ArgoUML

E.3.2. Reference Manual Structure

This section largely follows Chapter 4 of the existing User Manual, including the Gloassary. However it needs a few extra things.

  1. material on each of the diagram types, each of the artifacts that can appear on the diagrams and details of the features of each artifact type.

  2. An Index
