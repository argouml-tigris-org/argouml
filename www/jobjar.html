ArgoUML JobJar

Take on a Task

A lot of work has gone into developing ArgoUML, but much more still remains to be done.

Here is a list of pending tasks that you might be interested in doing. Each of these tasks would be a personally satisfying contribution toward making the ArgoUML vision a reality. Please join the developers mailing list (dev) to discuss tasks that you are interested in.

Special Note to Students

Each task could be made into an undergraduate term project or part of a masters degree project. Working on a free, open source software project such as ArgoUML can make your school project much more impressive, satisfying, and clarify potential benefits. Why start with nothing if you could build on the work of over a dozen other students? Why end your term project with just a grade when you could make a visible difference? The point of a term project is to get experience working on a "real" project. Using ArgoUML will give you experience with over 60,000 lines of Java code that implements a very real application.

Tasks

  • Use ArgoUML
    • Try out ArgoUML and make issues of things that are buggy or awkward.
    • Try to use it for a small design project. What is useful about the tool? What is needed?
    • Watch others use ArgoUML and report on how they use it.
  • UML Diagram
    • Implement a graph layout algorithm that takes into account the conventions of UML diagrams.
    • Implement arc routing that users find flexible and convient.
    • Implement the UML artifacts that ArgoUML still does not support, see UMLsupport.
    • Implement diagram types that are part of UML but not supported by ArgoUML yet, see UMLsupport.
    • Enable connection to class names to object names. Use case: When the user clicks on the properties pane for an object, the user has the option of choosing from a list of UML artifacts: classes, actors, etc.
    • Include edit commands under right click ...
    • Swim lanes for statechart and activity diagrams.
    • Implement printing of wall-sized diagram posters or design booklets.
    • For more ideas on diagram editing enhacements, see the GEF Web Site.
  • UML Meta-Model
    • Add an JDBC interface to the UML meta-model so that models can be stored in databases. (Work has already started on this.)
    • Add the ability to save the UML meta-model to a file using Java's ObjectSerialization library.
    • Add an external file format for compatability with other tools, to enhance ArgoUML's XMI support.
    • Generate an HTML reference web site for a UML model, much like JavaDoc does for Java code.
    • Recode parts of the meta-model to remove some current simplifying assumptions.
    • Enable data base modelling.
    • Enable import interface for XML documents.
  • Code Generation and Reverse Engineering
    • Implement code generation for an additional programming language.
    • Implement support for generating Java code that follows JavaBeans conventions.
    • Implement an extensible and customizable library for code generation in many languages.
    • Implement code generation for UML state machine models.
    • Generate assert() statements to check OCL constraints on a UML model at run-time.
    • Integrate a Java parser into ArgoUML to accept fragments of code entered in diagrams.
    • Use a Java parser to read code from files and construct a model.
    • Enable .jar reverse engineering. Perhaps using the parser that is available in the JDK distribution for generating javadocs from a jar?
    • Java RE to read package.html files in the source tree and add the information to MPackages. Java gereration to create a new package.html.
  • User Interface
    • Implement more table views of UML packages, classes, state machines, etc.
    • Improve any aspect of Argo's user interface and show that it is better.
    • Implement Undo.
    • Make ArgoUML work within Netbeans and Eclipse.
    • Add pop-up menus to several parts of Argo's interface.
    • Implement a vertically split explorer: lower half to show details of the upper selection.
    • Set the save as field when user selects File > "save graphics" the "file name" of the project" file of type" automatically i.e. "myproject.svg" for example.
  • Publishing the result
    • Implement a function to publish a complete view of the project in a set of graphically enhanced HTML file. This should include a choice of graphical output i.e. SVG, PNG, etc.
  • User Documentation
    • Write additions to the User's manual for ArgoUML, focus on a specific task or audience.
    • Suggest a new outline for a maintainable User's manual for ArgoUML.
    • Contribute questions and answers to the Users FAQ.
  • Documenation and Instructions for the developers of ArgoUML
    • Write additions to the Developer's Cookbook to help new people contribute to the ArgoUML project.
    • Write requirements / specification, e.g. how the yet unimplemented features should work in detail.
  • Object Constraint Language
    • Integrate the OCL parser into ArgoUML.
    • Enhance ArgoUML's OCL evaluator.
  • Cognitive Support for Design
    • Implement some critics or checklist items that give advice you think is useful.
    • Implement critics or checklists based on ideas from books or company policies.
    • Implement wizards to help fix problems.
  • Performance
    • Measure what parts of ArgoUML take time or space.
    • Enhance performace in some measurable way.
  • Testing
    • Implement test scripts using JUnit. Focus on a specific class or group of classes.

Suggest a New Task!
